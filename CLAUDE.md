# horde ‚Äî Project Conventions

## What this is

Cloud launcher for orc workflows. See SPEC.md for design, ORC_CONTRACT_EXPECTATIONS.md for the orc interface boundary.

## Language & Dependencies

- Go (latest stable), module: `github.com/jorge-barreto/horde`
- CLI: `github.com/urfave/cli/v3`
- SQLite: `github.com/mattn/go-sqlite3` (run history)
- AWS (v0.2): `github.com/aws/aws-sdk-go-v2` (ECS, SSM, S3)
- CDK construct (v0.2): TypeScript in `cdk/`, separate npm package `@horde/cdk`

## Commands

```bash
go build ./cmd/horde          # build
go test -short ./...           # unit tests only (fast, no Docker)
go vet ./...                   # lint

# Integration tests ‚Äî real Docker, real orc, real horde (~2 min)
go test -v -count=1 -timeout 10m ./test/integration/
```

## Git

- Commit messages: short and direct. No "Co-Authored-By", "Generated by", or any AI attribution lines.
- Stage specific files ‚Äî never `git add .` or `git add -A`.

## Code Style

- Standard Go conventions. `gofmt` and `go vet` clean.
- Errors are returned, not panicked. Wrap with context: `fmt.Errorf("loading config: %w", err)`.
- Interfaces defined in their own files (`store.go`, `provider.go`). Implementations in separate files (`sqlite.go`, `docker.go`, `ecs.go`).
- Tests next to implementation: `foo.go` ‚Üí `foo_test.go`.
- Never combine `t.Parallel()` with `t.Setenv`, `t.Chdir`, or `os.Setenv` ‚Äî `testing.T.Setenv` panics on tests with parallel ancestors. If a test needs env mutation, it cannot be parallel.

## Architecture

- `internal/provider/` ‚Äî Provider interface + implementations (docker; ecs in v0.2)
- `internal/store/` ‚Äî Store interface + SQLite (v0.1) and DynamoDB (v0.2) implementations
- `internal/config/` ‚Äî Config resolution (git remote, .env; SSM in v0.2)
- `internal/runid/` ‚Äî Run ID generation
- `cmd/horde/` ‚Äî CLI commands
- `docker/` ‚Äî Worker Dockerfile + entrypoint

## Key Design Decisions

- Git is a hard requirement ‚Äî repo URL inferred from local git remote
- v0.1 is zero-config: no config file, repo from git remote, secrets from `.env`, image defaults to `horde-worker:latest`
- SQLite for local run history (`~/.horde/horde.db`); DynamoDB for shared team history (v0.2)
- Store selection follows provider: docker ‚Üí SQLite, aws-ecs ‚Üí DynamoDB
- ECS provider (v0.2) calls RunTask directly ‚Äî no Lambda indirection. Infra config discovered via SSM at `/horde/config`
- EventBridge + status sync Lambda keeps DynamoDB accurate even when CLI disconnects (v0.2)
- maxConcurrent enforcement at launch time ‚Äî error, not queue (v0.2)
- Run timeout (default 24h) enforced by CLI timer (docker) and Fargate stopTimeout + Lambda (ECS)
- Provider.Launch returns `*LaunchResult` with `InstanceID` + `Metadata map[string]string` (no provider-specific fields on the generic type)
- Docker provider secrets loaded from `.env` file (gitignored) via `docker run --env-file`
- GIT_TOKEN protected via `GIT_ASKPASS` credential helper ‚Äî never in process args or `.git/config`
- `--json` flag on status/results/list for programmatic consumption (v0.2)
